## 计算机网络的分类—拓扑结构

当多个计算机互相连接在一起并形成网络时，这些计算机的连接结构就被称作拓扑结构。拓扑结构描述了计算机网络中各个节点之间物理连接的关系和方式。以下是常见的拓扑结构：

| 拓扑结构名称 | 特点                                                         |
| ------------ | ------------------------------------------------------------ |
| 总线拓扑     | 所有设备都连接到共同的电缆，数据通过广播传送                 |
| 星型拓扑     | 所有设备都连接到中心交换机，数据从源设备传输到目标设备经过中心交换机转发 |
| 环型拓扑     | 设备沿环路连接，相邻设备间直接通信，数据按顺序从源设备传输到目标设备 |
| 树型拓扑     | 由多个星型拓扑组合而成的树形结构，有效地减少了路由器的数量和长度 |
| 网状拓扑     | 设备通过多条不同的路径进行连接，提高了网络的可靠性和容错能力 |

每种拓扑结构都有其优点和缺点，其选择通常取决于具体的应用场景和需求。例如，总线拓扑结构适合小型局域网，但不适用于大型企业网络；在与外部网络连接时不适宜使用环型拓扑，而网状拓扑显然更为复杂。无论采用什么样的拓扑结构，都需要进行良好的规划、设计和管理，以确保网络的高性能、高可靠性和高安全性。

### 网络规划与设计—逻辑网络设计

逻辑网络设计是指在物理网络基础之上，进行更深入的网络规划与设计工作，从逻辑结构上对网络进行规划和设计。其目的是为了满足不同应用场景下的各种需求，如实现高效的路由、提高网络可靠性、减轻网络拥塞等。

在进行逻辑网络设计时，需要考虑以下方面：

1. IP 地址分配：确定每个设备所使用的 IP 地址，包括子网掩码、默认网关等信息，以确保设备能够正常通信。

2. 网络地址转换（NAT）：通过 NAT 技术将内部 IP 地址转换成公共 IP 地址，以增加网络安全性和扩大内部网络的访问范围。

3. 路由器配置：定义路由策略、优化路由表以提高路由效率，同时通过 VRRP 协议、HSRP 协议实现冗余路由器，确保网络的高可靠性和容错能力。

4. 交换机配置：配置 VLAN、VTP 等协议，实现广播域的隔离与管理。

5. 防火墙配置：通过防火墙策略，保护内部网络的安全，限制外部网络对内部网络的攻击，降低网络风险。

6. QoS 配置：通过 QoS 技术对网络带宽进行管理，实现流量控制、拥塞控制等功能。

在进行逻辑网络设计过程中，需要了解不同的网络协议和技术，同时匹配实际需求与预算。合理的逻辑网络设计能够提高网络的性能和安全性，为企业的信息化发展提供有力支持。

```C++
    1. 逻辑网络设计图

```







### 网络规划与设计—物理网络设计

物理网络设计是指在进行逻辑网络设计的基础上，考虑物理实现条件和传输介质选择等因素，根据具体实际情况和需求对网络设备、接口连接、布线走向进行规划和设计。其目的是为了确保网络的高效性、可靠性和易维护性。

在进行物理网络设计时，需要考虑以下方面：

1. 设备选型：包括路由器、交换机、防火墙、服务器等硬件设备的选型，确定功能、配置、品牌、型号等参数。

2. 接口连接：完成各个设备之间的接口连接，包括直连、跨网段连接、长距离光纤连接等，要注意选用高质量接口及优质网线，以避免因为接口不良所导致的网络故障。

3. 布线方式：根据实际情况选择水平式或垂直式布线，尽可能简化线路，提高网络美观性。

4. 机房环境：确定机房的位置和大小，考虑通风、温度、湿度、灰尘等因素，并配备相应设备和工具，如UPS电源、空调、静电地板等。

5. 网络安全：设置访问控制、加密传输、定期备份等措施，确保网络安全及数据完整。

物理设计过程中，通常需要多次调整和优化，特别是在设备选型和接口连接方面，需要充分考虑实际条件，精细化的操作能够让网络更加稳定高效。重要的是，一旦进行部署和建设，就需要严格遵守规划和设计，并对结果和细节进行后期监测和管理，以保证网络的正常运行和发展。

### 网络规划与设计--分层设计

![](https://gaidocs.oss-cn-hangzhou.aliyuncs.com/intenet/202305101433179.png)

分层设计是指在进行网络规划与设计时，将整个网络按功能和服务划分为若干独立的分层，在不同层之间建立适当的接口和协议，以利于模块化、可维护性和可扩展性。典型的分层设计模型包括三层模型和多层模型。

1. 三层模型
三层模型是指将整个网络分为核心层、汇聚层和接入层三层：

- 接入层：提供用户设备的连接，负责物理接口转换、端口隔离、VLAN 划分，可以使用交换机实现局域网互连。

- 汇聚层：承载较大量的流量，连接不同的接入层交换机，并进行路由控制和 QoS 策略，实现子网间互联。

- 核心层：提供高速中转能力，承载集中式应用，起到决策和控制作用。

该模型简单明了，容易理解和实施，其中每一层面向下层提供服务，面向上层调用服务，各层之间通过标准接口和协议相互关联。

2. 多层模型
多层模型是指在三层模型的基础上，根据实际需求和应用场景逐步增加分层次数，如企业级网络可分为五层模型：

- 物理层：提供物理接口和传输介质等硬件支持。

- 数据链路层：处理数据帧的格式和传输错误检测等工作，保证物理层传输的可靠性。

- 网络层：负责路由选择、分组交换、IP 地址分配等功能，实现子网间的通信和跨局域网通信。

- 传输层：提供端到端服务，提高数据传输的可靠性和效率，包括 TCP 和 UDP 两种协议。

- 应用层：面向用户提供各种应用服务，如 Web、电子邮件、文件共享等，包括 HTTP、SMTP、FTP 等协议。

根据不同需求和复杂度，还可以进一步扩展多层，如七层 OSI 参考模型。通过多层设计能够更好地对网络进行管理和维护，提高网络的安全性和稳定性。





### IP地址与子网划分

![](https://gaidocs.oss-cn-hangzhou.aliyuncs.com/intenet/202305101446582.png)



IP地址是计算机在Internet网络中的唯一标识，它由32位二进制数字组成，用点分十进制表示方式表达。在IPv4协议中，每个IP地址都包含一个网络部分和一个主机部分。而子网划分是指将一个IP地址空间进行更细致的划分，以获得更多可用的地址或者优化网络性能。

一般来说，IP地址分为A、B、C、D、E五类，其中A、B、C三类地址被广泛使用，而D、E地址则不常用。在这三类地址中，网络部分的长度是固定的，因此可以根据需要划分出不同数量的子网。以下是具体细节：

1. A类地址：默认使用8位的网络地址和24位的主机地址，可支持126个网络和每个网络16777214个主机。
2. B类地址：默认使用16位网络地址和16位主机地址，可支持16384个网络和每个网络65534个主机。
3. C类地址：默认使用24位网络地址和8位主机地址，可支持2097152个网络和每个网络254个主机。

对于较大的网络，需要更多的 IP 地址和更好的路由控制，因此我们需要使用子网划分和可变长度子网掩码（VLSM）技术来划分网络并获得更多的 IP 地址。

例如，在一个企业中，需要向100个子网分配IP地址，则可以使用VLSM和CIDR技术将B类地址的16位主机位划分为更小的子网（如18位网络位和14位主机位），从而得到更多可用的 IP 地址。

最后，对于任何网络地址规划，我们必须用正确的子网掩码来找出网络地址和主机地址。子网掩码是与IP地址相结合的32位数字，将IP地址按照网络位和主机位进行分隔。例如针对A类地址的子网掩码是255.0.0.0, 对C类地址的子网掩码就是255.255.255.0。

::: details 将B类IP地址168.195.0.0划分成27个子网，子网掩码为多少?

将B类IP地址168.195.0.0划分为27个子网，需要从主机位中借用5位二进制位进行划分，因为 $2^5=32 \geq 27$，可正好满足需要。得到划分后的子网掩码为255.255.255.224。

具体步骤如下：
1. 因为168.195是一个B类的IP地址，所以默认的子网掩码是255.255.0.0
2. 利用五个二进制位，可以表示出 $2^5 = 32$ 个不同的数字，因此我们可以将主机号划分为 $27$ 个 30（即24+5）位长度的网络和主机部分。
3. 然后我们可以确定第一个子网的网络地址为 168.195.0.0，扩大子网并递增网络号，即可得到其它子网的网络地址：168.195.0.32、168.195.0.64……依次类推，直到得到第27个子网。
4. 根据子网掩码的定义，子网掩码是一个32位的二进制数，其中前面存放的是网络地址的信息，而后面的是主机地址的信息。针对本题中的子网掩码，借用了5位主机位用于划分成27个子网，所以主机号中最后5位变成0，即为前面的指定网络位31个1，后面的域5个0，即11111111 11111111 11111111 11100000，或者写成255.255.255.224。

<font color="red">我们计算有几个子网，从左到右数$2^n$的几次方，也就是n的数值</font>

:::



::: details 例2，将B类IP地址168.195.0.0划分成若干子网每个子网内有主机700台，子网掩码为多少?

将B类IP地址168.195.0.0划分为若干个子网，每个子网内有 700 台主机，可以根据需要的主机数量确定所需的二进制位数。因为 $2^{10}=1024>700$，而 $2^9=512<700$，即主机号中需要用到至少 10 位二进制位来存储主机地址，而第一位是网络地址，因此需要借用 10 位二进制位作为主机地址，剩下的 22 位二进制位作为网络地址。因此，子网掩码为：255.255.252.0。

具体步骤如下：
1. 因为 168.195 是一个 B 类 IP 地址，其默认的子网掩码是 255.255.0.0，在此基础上进行划分。
2. 每个子网需要容纳 700 台主机，故需要用到 $log_2 (700) \approx 9.45$ 位的二进制位来存放主机地址。
3. 主机号中最高位（最左边）为 0，再结合借用的 9 位二进制位，所以我们使用 10 位二进制位来表示主机地址。
4. 子网掩码中，前面是网络部分，后面是主机部分：
   - 网络位需要 22 位来表示，即前 22 个二进制位全部为 1，
   - 主机位需要 10 位来表示，即后面 10 个二进制位全部为0。
5. 最终的子网掩码为255.255.252.0

<font color="red">我们计算子网中有几个主机，从右到左数$2^n$的几次方，也就是n的数值</font>
