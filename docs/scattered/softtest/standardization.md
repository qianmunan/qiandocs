## 软件标准化

::: tip

1. 国际标准：ISO、IEC等国际标准化组织
2. 国家标准：GB--中国，ANSI--美国，BS--英国，JIS--日本
3. 区域标准：又称为地区标准，如PASC--太平洋地区标准会议、CEN--欧洲标准委员会、ASAC--亚洲标准咨询委员会、ARSO--非洲地区标准化组织。
4. 行业标准：GJB--中国军用标准，MIT-S--美国军用标准，IEEE--美国电气电子工程师协会
5. 地方标准：国家的地方一级行政机构制定的标准
6. 企业标准
7. 项目规范

:::

软件标准化可以根据其对象、目的和内容等因素的不同进行分类，常见的软件标准化分类如下：

1. 功能安全标准：主要面向安全相关的软件产品或系统，如ISO 26262（适用于汽车领域的功能安全）、IEC 61508（适用于工业自动化系统）等。

2. 质量管理标准：通常是为了提高软件产品的质量水平，如ISO 9001（国际质量管理标准）、CMMI（能力成熟度模型集成）等。

3. 数据交换标准：为确保不同厂商间的软件能够正确地互操作，如XML（可扩展标记语言）、JSON（JavaScript对象表示法）等。

4. 码表标准：定义数据存储和传输中所使用的符号编码规范，如ASCII码（美国信息交换标准代码）、Unicode编码（统一字符编码）等。

5. 软件测试标准：涉及软件测试方法、测试过程和测试规范等方面，如IEEE 829（软件测试文档标准）、ISO/IEC 29119（软件测试标准）等。

6. 开放平台标准：提供软件开发者和用户之间的接口规范，以促进API（应用程序接口）的开发和普及，如RESTful API（一种基于Web的应用程序编程接口）等。



## 标准编号

1. 国际、国外标准代号：标准代号 + 专业类号 + 顺序号 + 年代号
2. 我国国家标准代号：
    1. 强制性标准代号为GB，
    2. 推荐性标准代号为GB/T
    3. 指导性标准代号为GB/Z
    4. 实物标准代号GSB
3. 行业标准代号：有汉语拼音大写字母组成（如电子行业为SJ）
4. 地方标准代号：由DB加上省级行政区代码的前两位
5. 企业标准代号：由Q加上企业代号组成
